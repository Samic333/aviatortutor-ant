## Core Logic & Relationships

### Identity & Roles
- **Legacy**: NextAuth with Role schema (STUDENT, INSTRUCTOR, ADMIN).
- **PHP**: `App\Lib\Auth` with session-based role checks and middleware-like role requirements in `index.php`.

### Booking Lifecycle
- **Step 1**: Student initiates booking via `create_booking.php`.
- **Step 2**: Stripe Checkout Session created.
- **Step 3**: On payment success, Webhook confirms booking (`status = 'CONFIRMED'`).
- **Step 4**: Instructor notified (via dashboard/logic).

### Profile Onboarding
- **Student**: Simple profile linked to `users`.
- **Instructor**: Detailed profile including bio, rating, and expertise. Requires admin approval to be "PUBLISHED".

### Database Performance
- Replaced Prisma ORM with lightweight **PDO** prepared statements.
- Added indexes on `user_id`, `instructor_id`, and `scheduled_time`.
- Optimized location-based and tag-based search using MySQL `JSON` functions where arrays were used in PG.
